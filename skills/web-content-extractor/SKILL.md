# 网页内容提取器

从任意URL提取正文内容，生成标题、摘要、关键点的结构化输出。

## 使用场景

✅ **适用**：
- 用户分享URL需要快速了解内容
- 需要将网页内容转为笔记素材
- 批量收集信息时需要结构化提取
- 研究调研时需要快速浏览多篇文章

❌ **不适用**：
- 需要登录才能访问的页面
- 动态JS渲染的内容（暂不支持）
- 视频/音频为主的页面

## 快速开始

```
用户：帮我提取这个网页 https://example.com/article
AI：[执行提取] → 返回标题、摘要、关键点
```

## 执行流程

1. **验证URL**
   - 检查是否为有效的HTTP/HTTPS链接
   - 如果无效，提示用户提供正确的URL

2. **获取网页内容**
   - 使用 `fetch_web` 工具获取网页文本
   - 如果失败（403/404/超时），如实告知用户

3. **提取结构化信息**
   - 提取标题（通常在 <title> 或 <h1>）
   - 提取正文核心内容（去除导航、广告、侧边栏）
   - 识别关键点（3-5个核心观点）

4. **生成输出**
   - 标题
   - 一句话摘要（50字内）
   - 关键点列表
   - 原文链接

## 输出格式

```markdown
## [标题]

**摘要**：[一句话概括]

**关键点**：
- 点1
- 点2
- 点3

**原文**：[URL]
**提取时间**：[时间戳]
```

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| URL无效 | 提示用户提供正确的URL |
| 网页无法访问 | 如实告知错误原因（403/404/超时） |
| 内容为空 | 提示可能是动态页面，建议直接阅读 |
| 内容过长 | 自动截取前3000字，提示"内容较长，已截取前部" |

## 注意事项

- 提取的内容不包含图片、视频等多媒体
- 对于付费内容或需要登录的页面，只能获取公开部分
- 建议用户验证关键信息，避免误解
